## 关键发现

**H1 · Agent 新架构**

Hermes Agent 的 Execute→Evaluate→Abstract→Refine 闭环是本轮最重要的架构参考。"技能不是设计出来的，而是进化出来的"——Agent 每次成功完成任务后，将执行路径结晶为可复用技能。GenericAgent 用 3.3K 行种子代码验证了这条路可行。

**H2 · Skill 系统设计**

SKILL.md 格式已统一，真正的问题从"格式"转向"内容设计"。Google ADK 总结了 5 种 Skill 内部逻辑模式：Tool Wrapper（知识封装）、Generator（结构生成）、Context Manager（上下文切换）、Chain of Skills（工作流组合）、Meta-Skill（技能创建技能）。最关键的机制：**渐进式披露**——先加载元数据，按需加载完整内容，减少 Token 消耗。

**H3 · 记忆系统**

从"chunk 存储"转向"fact-extraction"。用 LLM 从对话中主动抽取结构化事实（JSON），存向量库，检索时由 LLM 自己判断是否注入。关键洞察：**全量记录不如精准抽取**。

**H4 · 用户建模**

结构化特质 + 置信度 >> 原始文本堆叠。GetProfile 的思路：从对话中提取 typed facts（communication_style、expertise_level 等），带置信度评分。Personalization Editing 提供了另一个方向——不是注入偏好到 prompt，而是把偏好"编辑进"模型内部表示。

**H5 · 工具调用进化**

MCP（Model Context Protocol）已成 Agent 工具连接的"USB-C"接口。A2A 协议负责 Agent 间通信。OpenClaw 已支持 MCP，但暴露自定义 Skill 为 MCP server 尚未实践。

**H6 · 多 Agent 协作**

2026 是 Multi-Agent 元年。arXiv 综述提出了五维协作框架：Actor、Type（合作/竞争/竞合）、Structure、Strategy、Coordination Protocol。Hekate 目前是「发起者-执行者」的简单范式，缺乏显式角色定义和结构化通信协议。

**H7 · Self-Reflection**

Reflexion 的核心：反思结果用语言描述存入 episodic memory，供下次决策召回——不是更新权重，而是用语言反馈强化行为。Self-Refine（同模型迭代精炼）可立即集成，平均提升 20%。Gödel Agent 的运行时自我修改过于复杂，暂不适用。

**H8 · 评估体系**

**SWE-bench 被曝作弊事件**（Berkeley 团队用 10 行 Python 代码刷到满分，实际 0 bug 修复）是个警示：任何静态 benchmark 都会被钻空子。Hekate 需要的是**动态自我评估机制**，而非依赖外部排行榜。

---

## 对 Hekate 的启发

本轮研究指出了三个明确的改进方向：

1. **Skill 双层化**：在 TOOLS.md 的静态技能列表之上，增加 Persona 层（identity.yaml / rules.yaml）和 Memory 层（工作记录/偏好历史）
2. **记忆精准化**：从全量日志，转向 fact-extraction 的选择性抽取，每次会话后产出 3-5 条结构化洞察
3. **评估体系**：建立内部任务成功率追踪，记录每类任务的关键失误类型，作为能力成长的量化依据
